## Ход выполнения работы

### 1. Value Objects (Ценностные Объекты)

**Созданные Value Objects:**

1. **NoteId** - Уникальный идентификатор заметки.
   - Валидация: Проверка, что строка не пуста и имеет корректный формат (например, UUID).
   - Иммутабельность: ✅
   - Файл:src/domain/value_objects/note_id.py

2. **NoteTitle** - Заголовок заметки.
   - Валидация: Не может быть пустым, максимальная длина 255 символов.
   - Иммутабельность: ✅
   - Файл:src/domain/value_objects/note_title.py

3. **NoteContent** - Содержимое заметки.
   - Валидация: Проверка, что контент соответствует ожидаемому типу (например, текст). Может быть пустым, но неnull.
   - Иммутабельность: ✅
   - Файл:src/domain/value_objects/note_content.py

4. **OwnerId** - Идентификатор владельца заметки.
   - Валидация: Проверка, что строка не пуста.
   - Иммутабельность: ✅
   - Файл:src/domain/value_objects/owner_id.py

**Пример кода (NoteTitle):**
```javascript
// src/domain/value_objects/note-title.js
class NoteTitle {
  #value;
  static MAX_LENGTH = 255;

  constructor(title) {
    if (title === undefined || title === null || title.trim() === '') {
      throw new Error('Note title cannot be empty');
    }
    if (title.length > NoteTitle.MAX_LENGTH) {
      throw new Error(Note title cannot exceed ${NoteTitle.MAX_LENGTH} characters);
    }
    this.#value = title.trim();
  }

  get value() {
    return this.#value;
  }

  equals(other) {
    if (!(other instanceof NoteTitle)) return false;
    return this.#value === other.#value;
  }
}

module.exports = NoteTitle;
```

### 2. Entities (Сущности)

**Созданные Entity:**

1. **Note** - Основная сущность, представляющая заметку.
   - ID поле:id (типNoteId)
   - Бизнес-правила: Заметка имеет версию, которая увеличивается при изменении содержимого. Может быть удалена (мягкое удаление).
   - Файл:src/domain/entities/note.js

2. **Folder** - Папка для группировки заметок.
   - ID поле:id (типFolderId)
   - Бизнес-правила: Нельзя создать папку без имени. Нельзя переместить заметку в несуществующую папку.
   - Файл:src/domain/entities/folder.js

**Пример кода (Note):**
```javascript

// src/domain/entities/note.js
const NoteId = require('../value_objects/note_id');
const NoteTitle = require('../value_objects/note_title');
const NoteContent = require('../value_objects/note_content');
const OwnerId = require('../value_objects/owner_id');

class Note {
  #id;
  #ownerId;
  #title;
  #content;
  #version;
  #isDeleted;
  #createdAt;
  #updatedAt;

  constructor(id, ownerId, title, content) {
    if (!(id instanceof NoteId)) throw new Error('Invalid NoteId');
    if (!(ownerId instanceof OwnerId)) throw new Error('Invalid OwnerId');
    if (!(title instanceof NoteTitle)) throw new Error('Invalid NoteTitle');
    if (!(content instanceof NoteContent)) throw new Error('Invalid NoteContent');

    this.#id = id;
    this.#ownerId = ownerId;
    this.#title = title;
    this.#content = content;
    this.#version = 1;
    this.#isDeleted = false;
    this.#createdAt = new Date();
    this.#updatedAt = new Date();
  }

  // Геттеры
  get id() { return this.#id; }
  get ownerId() { return this.#ownerId; }
  get title() { return this.#title; }
  get content() { return this.#content; }
  get version() { return this.#version; }

  // --- Бизнес-методы ---
  updateContent(newContent) {
    if (!(newContent instanceof NoteContent)) {
      throw new Error('Invalid NoteContent');
    }
    if (this.#isDeleted) {
      throw new Error('Cannot update a deleted note');
    }
    this.#content = newContent;
    this.#version += 1;
    this.#updatedAt = new Date();
  }

  updateTitle(newTitle) {
    if (!(newTitle instanceof NoteTitle)) {
      throw new Error('Invalid NoteTitle');
    }
    if (this.#isDeleted) {
      throw new Error('Cannot update a deleted note');
    }
    this.#title = newTitle;
    this.#updatedAt = new Date();
  }

  delete() {
    if (this.#isDeleted) {
      throw new Error('Note is already deleted');
    }
    this.#isDeleted = true;
    this.#updatedAt = new Date();
  }

  restore() {
    if (!this.#isDeleted) {
      throw new Error('Note is not deleted');
    }
    this.#isDeleted = false;
    this.#updatedAt = new Date();
  }

  // Реализация равенства по ID
  equals(other) {
    if (!(other instanceof Note)) return false;
    return this.#id.equals(other.#id);
  }

  // hashCode для использования в коллекциях
  hashCode() {
    return this.#id.value.hashCode(); // упрощенная реализация
  }
}

module.exports = Note;
```

### 3. Aggregate Root (Корневой агрегат)

**Aggregate Root:**Note (Заметка)

**Границы агрегата:**
- Корень:Note (сущность)
- Внутренние сущности: Нет (у заметки нет внутренних сущностей, она самодостаточна)
- Value Objects:NoteId,NoteTitle,NoteContent,OwnerId

**Инварианты агрегата:**

| № | Инвариант | Как проверяется |
|---|----------|----------------|
| 1 | Заметка не может быть пустой (без заголовка и содержания одновременно). | В конструкторе проверяется, что переданные VO корректны. Если бизнес-правило требует наличия хотя бы одного поля, добавляется проверкаif (title.value === '' && content.value === '') { throw ... } |
| 2 | Нельзя изменить удаленную заметку. | В методахupdateTitle,updateContent,delete проверяется флагisDeleted. |
| 3 | Версия заметки увеличивается при каждом изменении контента. | В методеupdateContent инкрементируетсяversion. |
| 4 | Владелец заметки является обязательным и не может быть изменен после создания. | В конструкторе проверяетсяownerId, и нет публичного метода для его изменения. |

**Пример кода Aggregate Root (метод updateContent с инвариантами):**
```javascript
// src/domain/entities/note.js (фрагмент)
updateContent(newContent) {
  // Инвариант #2: Проверка, что заметка не удалена
  if (this.#isDeleted) {
    throw new Error('Cannot update a deleted note');
  }
  // Инвариант #1 (частично): Проверка, что newContent - корректный VO
  if (!(newContent instanceof NoteContent)) {
    throw new Error('Invalid NoteContent');
  }
  
  // Инвариант #3: Увеличение версии при изменении
  this.#content = newContent;
  this.#version += 1;
  this.#updatedAt = new Date();
}
```

### 4. Domain Events (Доменные события)

**Созданные события:**

1. **NoteCreated** - Генерируется при создании новой заметки.
   - Данные:noteId,ownerId,title,occurredAt
   - Файл:src/domain/events/note-created.js

2. **NoteContentUpdated** - Генерируется при изменении содержимого заметки.
   - Данные:noteId,newVersion,occurredAt
   - Файл:src/domain/events/note-content-updated.js

3. **NoteDeleted** - Генерируется при удалении заметки.
   - Данные:noteId,occurredAt
   - Файл:src/domain/events/note-deleted.js

**Пример кода события (NoteCreated):**
```javascript
// src/domain/events/note-created.js
class NoteCreated {
  #noteId;
  #ownerId;
  #title;
  #occurredAt;

  constructor(noteId, ownerId, title) {
    this.#noteId = noteId;
    this.#ownerId = ownerId;
    this.#title = title;
    this.#occurredAt = new Date();
  }

  get noteId() { return this.#noteId; }
  get ownerId() { return this.#ownerId; }
  get title() { return this.#title; }
  get occurredAt() { return this.#occurredAt; }
}

module.exports = NoteCreated;
```

**Регистрация событий внутри агрегата:**
Для этого агрегат должен иметь коллекцию для хранения событий, которые произошли во время его жизненного цикла.
```javascript
// src/domain/entities/note.js (добавление коллекции событий)
class Note {
  #events = [];

  constructor(id, ownerId, title, content) {
    // ... существующая логика ...
    this.#registerEvent(new NoteCreated(id, ownerId, title));
  }

  updateContent(newContent) {
    // ... валидация ...
    this.#registerEvent(new NoteContentUpdated(this.#id, this.#version));
  }

  delete() {
    // ... валидация ...
    this.#registerEvent(new NoteDeleted(this.#id));
  }

  #registerEvent(event) {
    this.#events.push(event);
  }

  pullEvents() {
    const events = [...this.#events];
    this.#events = [];
    return events;
  }
}
```


## Контрольные вопросы

1.  **В чём отличие Value Object от Entity?**
    *   **Entity** имеет уникальный идентификатор (ID), который остается неизменным на протяжении всей жизни объекта. Две сущности с разными ID считаются разными, даже если все их поля совпадают. Она изменяема (mutable).
    *   **Value Object** не имеет собственного ID. Он полностью определяется своими атрибутами. Два Value Object с одинаковыми значениями полей считаются идентичными. Он иммутабелен (immutable). Любое изменение создает новый экземпляр.

2.  **Почему Aggregate Root должен инкапсулировать доступ к внутренним сущностям?**
    *   Чтобы обеспечить соблюдение бизнес-инвариантов (правил) для всей группы объектов (агрегата). Если бы внешний код мог напрямую изменять внутренние сущности, он мог бы нарушить эти правила. Aggregate Root выступает единой точкой входа для всех изменений, гарантируя, что агрегат всегда находится в консистентном состоянии.

3.  **Какая роль Domain Events? Приведите пример из вашей системы.**
    *   Domain Events используются для уведомления других частей системы (или других сервисов) о том, что в домене произошло важное событие. Они позволяют реализовать слабую связанность между компонентами. **Пример:** Когда заметка обновляется (NoteContentUpdated), это событие может быть обработано другим сервисом (например,SyncService), чтобы инициировать синхронизацию этой заметки на всех устройствах пользователя.

4.  **Как вы проверяете инварианты в вашем агрегате? Приведите пример.**
    *   Инварианты проверяются в публичных методах агрегата перед изменением его состояния. В моей системе, в методеNote.updateContent(), я проверяю инвариант "Нельзя изменить удаленную заметку". Еслиthis.#isDeleted === true, метод выбрасывает исключениеError('Cannot update a deleted note'), предотвращая недопустимое изменение состояния.

5.  **Почему Value Objects делаются иммутабельными?**
    *   **Безопасность:** Поскольку у VO нет ID, если бы он был изменяемым, то изменение одного экземпляра могло бы случайно затронуть все ссылки на него, что привело бы к непредсказуемому поведению и ошибкам.
    *   **Простота:** Иммутабельные объекты проще в понимании, тестировании и использовании, особенно в многопоточных средах или в комбинации с коллекциями (например, вSet или как ключи вMap).
