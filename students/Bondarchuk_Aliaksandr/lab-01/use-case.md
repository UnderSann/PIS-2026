## Use-case: Создание новой заметки из локального .txt файла

**Первичный актор**: Зарегистрированный пользователь системы Notes Sync.

**Цель**: Пользователь хочет создать новую заметку, загрузив существующий текстовый файл со своего устройства, чтобы затем иметь к ней доступ и возможность редактирования с любого другого устройства.

**Предусловия**:
- Пользователь успешно авторизован в веб-интерфейсе системы.
- На локальном устройстве пользователя существует текстовый файл с расширением `.txt`.
- У пользователя есть активное интернет-соединение.

**Основной поток** (Happy Path):
1.  Пользователь нажимает кнопку "Создать заметку" или "Загрузить файл" в веб-интерфейсе.
2.  Система отображает диалог выбора файла.
3.  Пользователь выбирает нужный `.txt` файл в проводнике своей ОС и подтверждает выбор.
4.  Система (UI) отправляет файл и метаданные (название, дата создания) на сервер (`Notes API`).
5.  `Notes API` принимает файл и сохраняет его в постоянное хранилище (File Storage).
6.  `Notes API` создаёт запись о новой заметке в базе данных (БД), связывая её с пользователем и путём к файлу. Заметке присваивается уникальный идентификатор.
7.  `Notes API` вызывает `Version Service` для создания первой версии (v1) этой заметки, фиксируя текущее состояние файла.
8.  `Notes API` отправляет событие в `Sync Service` о том, что для пользователя появилась новая заметка, и её нужно синхронизировать со всеми его устройствами.
9.  Система (UI) отображает пользователю сообщение об успешном создании заметки и её появлении в общем списке.

**Постусловия**:
- В системе создана новая заметка, привязанная к аккаунту пользователя.
- Файл заметки сохранён в защищённом хранилище.
- Создана начальная версия заметки в истории версий.
- Сгенерировано событие для синхронизации данной заметки на другие устройства пользователя.

**Альтернативные потоки**:
- 3a. **Пользователь выбрал файл неверного формата.**
  - 3a1. Система проверяет расширение файла и определяет, что оно не `.txt`.
  - 3a2. Система немедленно уведомляет пользователя об ошибке: "Можно загружать только файлы .txt".
  - 3a3. Диалог выбора файла остаётся открытым для повторной попытки. Use-case возвращается к шагу 2.

**Исключительные ситуации**:
- 5a. **Ошибка сохранения файла в File Storage (например, диск переполнен).**
  - 5a1. `Notes API` ловит исключение от File Storage.
  - 5a2. `Notes API` возвращает ошибку 500 (Internal Server Error) в UI.
  - 5a3. Система (UI) отображает пользователю сообщение: "Не удалось сохранить файл. Попробуйте позже или обратитесь в поддержку".
  - 5a4. Use-case завершается неудачей. Запись в БД не создаётся.
- 7a. **Сервис версионирования временно недоступен.**
  - 7a1. `Notes API` не получает ответ от `Version Service` в течение таймаута (например, 3 секунды).
  - 7a2. **Компенсация:** `Notes API` НЕ откатывает создание заметки и файла. Вместо этого он помещает задание на создание версии в отдельную очередь (Queue) для повторной попытки.
  - 7a3. Заметка в БД получает флаг `versioning_pending = true`.
  - 7a4. `Notes API` возвращает в UI успешный ответ (200 OK), но с предупреждением в теле ответа: "Заметка сохранена. История версий будет создана автоматически через несколько минут".
  - 7a5. Use-case считается успешно завершённым с отложенным действием.
