## Ход выполнения работы

### 1. Команды (Commands)

**Созданные команды:**

1. **CreateNoteCommand** - Создание новой заметки.
   - Поля: ownerId, title, content
   - Валидация: Проверка, что ownerId не пуст, title не пуст и не превышает 255 символов.
   - Файл: application/command/create_note_command.js

2. **UpdateNoteContentCommand** - Обновление содержимого заметки.
   - Поля: noteId, newContent
   - Валидация: Проверка, что noteId имеет корректный формат, newContent не null.
   - Файл: application/command/update_note_content_command.js

3. **DeleteNoteCommand** - Мягкое удаление заметки.
   - Поля: noteId
   - Валидация: Проверка, что noteId существует.
   - Файл: application/command/delete_note_command.js

**Пример кода команды:**
```javascript
// application/command/create_note_command.js
class CreateNoteCommand {
  #ownerId;
  #title;
  #content;

  constructor(ownerId, title, content) {
    // Валидация примитивов
    if (!ownerId || typeof ownerId !== 'string' || ownerId.trim() === '') {
      throw new Error('OwnerId must be a non-empty string');
    }
    if (!title || typeof title !== 'string' || title.trim() === '') {
      throw new Error('Title must be a non-empty string');
    }
    if (title.length > 255) {
      throw new Error('Title must not exceed 255 characters');
    }
    if (content !== undefined && content !== null && typeof content !== 'string') {
      throw new Error('Content must be a string if provided');
    }

    this.#ownerId = ownerId;
    this.#title = title;
    this.#content = content || '';
  }

  get ownerId() { return this.#ownerId; }
  get title() { return this.#title; }
  get content() { return this.#content; }
}

module.exports = CreateNoteCommand;
```

### 2. Command Handlers

**Созданные обработчики:**

1. **CreateNoteHandler** - Обрабатывает создание заметки.
   - Шаги обработки:
     1. Преобразовать примитивы команды в Value Objects.
     2. Создать агрегат Note.
     3. Сохранить агрегат через репозиторий.
     4. Публиковать доменные события.
   - Возвращает: ID заметки.
   - Файл: application/command/handlers/create_note_handler.js

2. **UpdateNoteContentHandler** - Обновляет содержимое заметки.
   - Шаги: Загрузить заметку, проверить владельца, обновить содержимое, сохранить.
   - Файл: application/command/handlers/update_note_content_handler.js

3. **DeleteNoteHandler** - Удаляет заметку.
   - Шаги: Загрузить заметку, вызвать метод delete(), сохранить.
   - Файл: application/command/handlers/delete_note_handler.js

**Пример кода handler:**
```javascript
// application/command/handlers/create_note_handler.js
const Note = require('../../../domain/entities/note');
const NoteId = require('../../../domain/value_objects/note_id');
const NoteTitle = require('../../../domain/value_objects/note_title');
const NoteContent = require('../../../domain/value_objects/note_content');
const OwnerId = require('../../../domain/value_objects/owner_id');

class CreateNoteHandler {
  #noteRepository;
  #eventBus;

  constructor(noteRepository, eventBus) {
    this.#noteRepository = noteRepository;
    this.#eventBus = eventBus;
  }

  async handle(command) {
    // 1. Создание Value Objects из примитивов команды
    const noteId = new NoteId(crypto.randomUUID()); // Генерация ID
    const ownerId = new OwnerId(command.ownerId);
    const title = new NoteTitle(command.title);
    const content = new NoteContent(command.content);

    // 2. Создание агрегата
    const note = new Note(noteId, ownerId, title, content);

    // 3. Сохранение через репозиторий
    await this.#noteRepository.save(note);

    // 4. Публикация доменных событий
    const events = note.pullEvents();
    for (const event of events) {
      await this.#eventBus.publish(event);
    }

    // 5. Возврат ID заметки (следуя CQRS)
    return noteId.value;
  }
}

module.exports = CreateNoteHandler;
```

### 4. Query Handlers

**Созданные обработчики запросов:**

1. **GetNoteByIdHandler** - Получает заметку по ID.
   - Репозиторий: NoteRepository
   - Возвращает: NoteDto
   - Обработка ошибки: если заметка не найдена, возвращает null или выбрасывает NotFoundError.
   - Файл: application/query/handlers/get_note_by_id_handler.js

2. **ListNotesByOwnerHandler** - Получает все заметки владельца.
   - Репозиторий: NoteRepository
   - Возвращает: Array<NoteDto>
   - Файл: application/query/handlers/list_notes_by_owner_handler.js

3. **GetSyncStatusHandler** - Вычисляет статус синхронизации.
   - Репозиторий: NoteRepository
   - Возвращает: SyncStatusDto
   - Файл: application/query/handlers/get_sync_status_handler.js

**Пример кода:**
```javascript
// application/query/handlers/get_note_by_id_handler.js
const NoteId = require('../../../domain/value_objects/note_id');
const NoteDto = require('../dto/note_dto');

class GetNoteByIdHandler {
  #noteRepository;

  constructor(noteRepository) {
    this.#noteRepository = noteRepository;
  }

  async handle(query) {
    // 1. Преобразование в Value Object
    const noteId = new NoteId(query.noteId);

    // 2. Поиск в репозитории
    const note = await this.#noteRepository.findById(noteId);
    if (!note) {
      return null; // Или throw new NotFoundError('Note not found')
    }

    // 3. Преобразование в DTO для чтения
    return new NoteDto(
      note.id.value,
      note.ownerId.value,
      note.title.value,
      note.content.value,
      note.version,
      note.isDeleted,
      note.createdAt,
      note.updatedAt
    );
  }
}

module.exports = GetNoteByIdHandler;
```

## Контрольные вопросы

1. **В чём разница между Command и Query?**
   - **Command** изменяет состояние системы. Она выполняет действие (создание, обновление, удаление) и обычно не возвращает данные (или возвращает только ID). Query **только читает** данные, не изменяя состояние системы, и возвращает DTO.

2. **Почему Command Handler возвращает только ID, а не весь объект?**
   - Чтобы **избежать утечки доменной модели** наружу и следовать принципу CQRS. После выполнения команды клиент должен сделать отдельный GET-запрос, чтобы получить актуальные данные. Это также улучшает тестируемость и разделение ответственности.

3. **Где должна выполняться валидация: в команде, обработчике или доменной модели?**
   - **Примитивная валидация** (NotBlank, Positive, формат) - в команде или обработчике. Это защита от неверных входных данных.
   - **Бизнес-инварианты** (например, "нельзя удалить заметку, если она уже удалена") - в доменной модели. Это гарантирует корректность состояния агрегата.

4. **Можно ли вызывать Query из Command Handler?**
   - Технически да, но **не рекомендуется**. Это может нарушить CQRS и привести к сложностям в тестировании. Лучше загружать необходимые данные через репозиторий в самом обработчике, не используя Query Handlers.

5. **Зачем разделять Request DTO (от клиента) и Command (внутренний)?**
   - **Request DTO** (например, HTTP body) зависит от формата API (JSON, XML) и может содержать поля для сериализации.
   - **Command** - это внутренняя структура приложения, которая может быть более строгой и валидированной. Разделение позволяет **независимо менять API** и бизнес-логику, не затрагивая друг друга.
